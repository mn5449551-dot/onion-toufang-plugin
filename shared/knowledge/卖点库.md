## 使用原则

- 先选功能，再选卖点。每条方向选 1-3 个互补卖点，不把同一功能下的所有卖点全塞进去。
- 卖点是生成策略，不一定要原样出现在画面文字里；画面文案要转成具体学生/家长场景。
- 用户给的新卖点不在库里时，先判断是否是新事实、旧事实的新说法、还是业务越界；不直接写成既有能力。
- 当前 Base 的 `功能` 选项可能尚未覆盖全部功能。写入时以真实 Base 选项为准；若选项未扩展，先用 `其他` 或维护人指定的映射，不让 Skill 静默改 schema。
- `学习机` 是渠道/竞品/使用场景，不是独立 APP 功能；用户说“学习机素材”时，要落回同步课、动画视频课、总复习、拍题精学等具体能力。

## F001 动画视频课

| 卖点 | 适合怎么打 | 典型场景 |
|---|---|---|
| 有趣易懂，愿意学能学会 | 把“学不进去”转成“孩子愿意主动点开看” | 厌学、畏难、对数学/物理没兴趣 |
| 5-8 分钟短时高频 | 强调碎片时间也能学完一个知识点，降低坚持门槛 | 睡前、通勤、饭后、课间复习 |
| 抽象知识可视化 | 用动画或实验把概念变成能看见的画面 | 几何、函数、力学、电路、化学反应 |
| 自主学习能力 | 孩子自己安排学习、自己完成看课和练习 | 家长忙、孩子需要培养自主性 |
| 课程打磨重 | 每节课经过教研、讲稿、教具和动画制作等环节 | 家长关心课程质量和专业性 |

## F002 同步课

| 卖点 | 适合怎么打 | 典型场景 |
|---|---|---|
| 教材同步 | 对齐校内进度，课前预习、课后复习都能接上 | “今天课堂没听懂”“明天要讲新课” |
| 覆盖广泛 | 小学/初中/高中多学科、多教材版本 | 不同地区、不同教材用户 |
| 概念课+解题课 | 先听懂知识，再学怎么做题 | 基础概念不清、题型不会迁移 |
| 学完马上练 | 看课后配套练习，马上验证掌握程度 | 学完就忘、家长不知道有没有学会 |
| 低起点梯度走 | 由易到难，先建立信心再提升 | 基础薄弱、害怕做题 |

|---|---|---|

| 拓展拔高 | 不只补基础，也给中上游学生做能力提升 | 压轴题、综合题、想冲高分 |

## F004 拍题精学

| 卖点 | 适合怎么打 | 典型场景 |
|---|---|---|
| 一拍秒出答案解析 | 快速识别题目，给答案和完整解析 | 作业卡壳、赶时间核对 |
| 识别准确答案靠谱 | 解决反复拍题、题目识别错、答案不可信的问题 | “拍了改、改了拍”用户 |
| 得分点拆解 | 把关键步骤拆成阅卷能给分的点 | 备考、步骤分丢分、规范书写 |
| 步骤详细不跳步 | 每一步拆开讲，不只给一坨解析 | 基础弱、看答案也看不懂 |
| AI交互式精学 | 不懂可继续追问，帮助从“看答案”变成“会解题” | 自学无人答疑、解析跳步 |
| 思路骨架/关键突破口 | 不直接替孩子抄答案，而是提示思路和突破口 | 家长担心拍题变抄题 |
| 闭环通关学一类题 | 定位卡点、讲透知识、同类题验证 | 换一道题又不会 |
| 讲解生动有趣 | 少用密密麻麻文字解析，用更像老师讲题的方式 | 孩子排斥普通答案软件 |

不要默认写“语音讲解”“自动批改作文”等未确认能力；只有用户明确确认产品已上线时才能使用。

## 通用背书与差异化

| 类型 | 可用卖点 | 适合怎么打 |

## 生成时的判断顺序

1. 用户指定功能：只在该功能和相关通用背书里选卖点。
2. 用户只给节点：根据时间节点择优表先定功能，再挑卖点。
3. 用户给人群或痛点：优先匹配能解决该痛点的功能，不为了覆盖功能库而分散。
4. 批量生成 5 条以上：卖点、场景、人群、视觉钩子至少拆成多个组合，不把同一卖点换皮重复。
5. 写 Base：`功能`、`卖点` 字段必须符合当前表字段选项；新增选项由维护人同步，失败记录进 pending，不丢数据。
