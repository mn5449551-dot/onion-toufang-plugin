# 环境自检清单

> onion-help 的诊断参考。按用户意图选择检查项；单项失败不应阻止其它本地检查继续执行。

## 判定口径

| 标记 | 含义 |
|---|---|
| ✅ | 已满足，核心 skill 可继续使用 |
| ⚠️ | 有风险或建议修复，但不一定阻塞 |
| 🔴 | 阻塞项，需要先处理 |

## 1. lark-cli 与 Base 权限

```bash
which lark-cli
set -a; source ~/.onion-ad/.env 2>/dev/null; set +a
BASE="${ONION_BASE_APP_TOKEN:-WIoGb0ksnaREvJsPtQCcW8Lsnfg}"
lark-cli base +base-get --base-token "$BASE" --as user 2>&1 | head -10
```

期望：能找到 `lark-cli`，且 Base 返回成功，输出里的 `identity` 是 `user`。这关系到 Base 的 `创建人` 系统字段能否记录真实团队成员。

失败处理：

- 找不到命令：按本机 lark-cli 安装说明补装。
- 401：执行 `lark-cli auth login`，用飞书账号扫码。
- 403：请徐豪把当前飞书账号加入生产 Base 协作者，权限至少可编辑。
- `identity` 不是 `user`：重新用 `--as user` 检查；不要让核心写入走 bot 身份。
- 超时：检查网络、公司 VPN 或代理。

## 2. `.env` 配置

```bash
ls -la ~/.onion-ad/.env 2>&1
stat -f "%A" ~/.onion-ad/.env 2>/dev/null
grep "^LAOZHANG_API_KEY=" ~/.onion-ad/.env | grep -v "sk-你的\\|sk-xxx" | head -1
grep "^ONION_BASE_APP_TOKEN=" ~/.onion-ad/.env | grep -v "^ONION_BASE_APP_TOKEN=$" | head -1
```

判定：

- 文件不存在：🔴，复制模板后补 key。
- `LAOZHANG_API_KEY` 缺失或仍是占位符：🔴，出图会失败。
- 权限不是 `600`：⚠️，建议 `chmod 600 ~/.onion-ad/.env`。
- Base token 缺失：⚠️，共享脚本有默认 token，但建议同步模板。

修复模板：

```bash
PLUGIN_ROOT=/path/to/onion-toufang-plugin
mkdir -p ~/.onion-ad
cp "$PLUGIN_ROOT/.env.template" ~/.onion-ad/.env
vim ~/.onion-ad/.env
chmod 600 ~/.onion-ad/.env
```

## 4. 共享脚本 dry-run

```bash
PLUGIN_ROOT=/path/to/onion-toufang-plugin
python3 "$PLUGIN_ROOT/shared/scripts/write_record.py" \
  --table-id tblLWPSHrZT95oy7 \
  --records '[{"fields":{"素材方向":"自检方向","状态":"待用"}}]' \
  --dry-run

python3 "$PLUGIN_ROOT/shared/scripts/update_status.py" \
  --table-id tblLWPSHrZT95oy7 \
  --record-id recSELF_CHECK \
  --status 已用 \
  --dry-run

python3 "$PLUGIN_ROOT/shared/scripts/write_image_group.py" \
  --copy-id recCOPY_SELF_CHECK \
  --images '["/tmp/onion-self-check.png"]' \
  --metadata '{"图片形式":"单图","状态":"待用"}' \
  --dry-run
```

期望：只输出 `dry_run: true` 和将要提交的 payload，不真实写 Base。

失败处理：若 JSON 参数解析失败，优先检查 shell 引号；若 import 或路径失败，回到第 3 项修脚本。

## 7. 老张 API 连通性

只在用户怀疑出图失败、key 失效、或环境完整体检时执行。

```bash
set -a; source ~/.onion-ad/.env 2>/dev/null; set +a
KEY="${LAOZHANG_API_KEY:-EMPTY}"
[ "$KEY" = "EMPTY" ] && echo "LAOZHANG_API_KEY missing" && exit 1
curl -sS -w "\n[HTTP %{http_code}]" \
  -H "Authorization: Bearer $KEY" \
  "https://api.laozhang.ai/v1/models" 2>&1 | head -5
```

判定：

- 200：✅ key 和网络基本可用。
- 401：🔴 key 过期或填错。
- 429/余额相关：🔴 额度或频率问题。
- 网络失败：🔴 检查代理或 VPN。

## 8. pending 队列

```bash
python3 - <<'PY'
import json
from pathlib import Path

path = Path.home() / ".onion-ad" / "pending.jsonl"
if not path.exists() or path.stat().st_size == 0:
    print("pending_total=0 ambiguous=0 retry_ge_3=0")
    raise SystemExit

total = ambiguous = retry_ge_3 = 0
bad_lines = 0
for line in path.read_text(encoding="utf-8").splitlines():
    if not line.strip():
        continue
    try:
        item = json.loads(line)
    except json.JSONDecodeError:
        bad_lines += 1
        continue
    total += 1
    ambiguous += int(bool(item.get("ambiguous")))
    retry_ge_3 += int(int(item.get("retry_count", 0)) >= 3)

print(f"pending_total={total} ambiguous={ambiguous} retry_ge_3={retry_ge_3} bad_lines={bad_lines}")
PY
```

判定：

- `pending_total=0`：✅。
- 有 pending 但 `retry_ge_3=0` 且 `ambiguous=0`：⚠️，后续可安全重试。
- `ambiguous>0`：🔴 或 ⚠️，取决于用户是否要继续写入；必须先人工核对 Base 是否已写成功。
- `bad_lines>0`：🔴，队列文件可能损坏。

不要自动执行 `retry_pending.py`。只有用户明确要求重试，且已处理 ambiguous 项后，才运行：

```bash
PLUGIN_ROOT=/path/to/onion-toufang-plugin
python3 "$PLUGIN_ROOT/shared/scripts/retry_pending.py"
```
