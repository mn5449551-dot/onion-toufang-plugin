# 系列一致性

本文件说明如何把 `shared/recipes/branch-prompt.md` 的 SAME/CHANGE 模型用于已有图组迭代。

## 架构差异

| 场景 | 参考图 | CHANGE 来源 |
|---|---|---|
| 同套双/三图 branch | 刚生成的 `setN_img1.png` | 同套文案的下一句 |
| 基于旧图组迭代 | 原图组的图1或用户上传图 | 用户指定的力度和改动轴 |

写法仍然使用 `shared/recipes/branch-prompt.md`：默认锁定场景、人物、构图、风格、光线、字体、Logo/CTA、排版节奏，只改用户指定的部分。

## SAME 解锁

| 轴 | 解锁项 | 说明 |
|---|---|---|
| IP | 人物形象 | 只换角色时，其余 SAME 继续锁 |
| 场景细节 | 局部场景元素 | 不等于换完整叙事 |
| 比例 | 构图比例 | 常见于渠道适配 |
| Logo / CTA | Logo 或按钮区域 | 只改对应区域 |
| 风格参考 | 风格和色调 | 需要用户明确选择 |
| 图片形式 | 不适用 SAME | 单/双/三图变化属于换形式 |

不要主动解锁用户没要求的轴。

## Reference 规则

默认只传原图 PNG。用户明确换 IP、Logo 或风格时，才额外传新资产参考图。

永远不要默认重传字体参考；字体应从原图继承。

## 调度

按 `shared/recipes/render-chain.md` 的 `render.py` 接口执行。基于旧图生成新图时，把旧图 PNG 当作 branch reference。

如果原图组是双/三图且新图组保持同形式：

1. 新图1基于原图1做 delta。
2. 新图2/3基于新图1做 delta。

这样新图组内部仍然自洽，而不是每一张都各自贴着旧图漂移。

## 决策检查表（内部）

Use this checklist privately. Only summarize issues to the user if asked why.

- 新图是否看得出和原图同源。
- 只改了用户指定的轴。
- 同套图内 IP、字体、Logo 和色调一致。
- 情绪没有从正向回退到强负面。
- 换形式时没有强行套 branch SAME。
