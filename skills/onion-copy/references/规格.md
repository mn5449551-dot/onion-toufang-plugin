# 文案规格

本文件说明单图、双图、三图分别需要什么字段，以及写多套时如何保持差异。字数是视觉建议，不是硬卡；硬标准是通顺、能读懂、能入图。

## 通用决策

文案先服从方向卡，再服从渠道和图片形式。

| 判断 | 原则 |
|---|---|
| 方向心智 | 文案必须继承方向卡的场景问题、差异化解法和奇效 |
| 人群视角 | 用户不指定人群时，默认学生视角 |
| 渠道语气 | 信息流要快抓注意力；应用商店/学习机单图要口语、清楚、有画面感 |
| 图片形式 | 单图一句话说完；双图形成前后关系；三图形成节奏 |
| 多套差异 | 3 套及以上至少覆盖 3 类切入角度；5 套同一类型最多 2 套 |

常用切入角度：钩子、共情、数字证据、反差对比、场景共鸣。它们是参考维度，不是必须机械选槽；渠道不匹配时宁可换类型，不要为了凑数牺牲转化。

内部先建立“功能 × 用户”理解：具体场景、具体情绪、产品动作和可感知变化都要来自当前功能事实。单图、双图、三图再选择各自最适合的表达关系；不要把某一种关系当成所有形式的默认模板。

## 决策检查表（内部）

Use this checklist privately. Only summarize issues to the user if asked why.

- 是否围绕方向卡心智，而不是换了另一个卖点。
- 多套之间是否只是同义改写。
- 每句话读起来是否像真人说话，而不是关键词标签云。
- 默认学生视角是否成立：不是家长腔、品牌腔或销售说明书。
- 产品动作是否清楚：用户能否看懂洋葱具体做了什么。
- 是否把产品事实写错位，例如把私教班事实写进拍题精学。

## 单图

单图 = `主标题 + 副标题`。它只有一张图，必须一句话讲完痛点和解法。

| 字段 | 作用 | 建议 |
|---|---|---|
| 主标题 | 抓痛点、钩子或核心卖点 | 信息流 8-14 字；应用商店/学习机 6-10 字 |
| 副标题 | 补解法、产品主体和收益 | 14-22 字左右，通顺优先 |

写法：

- 信息流主标题可以更冲突，但不要变成夸张恐吓。
- 应用商店和学习机单图共用逻辑：默认学生视角，有具体场景，产品动作清楚。
- 应用商店用户在主动比较 App，主副标优先回答“干什么、凭什么、为什么下你”；学习机用户也按同一单图逻辑写。
- 副标题不是堆功能，应该补足“为什么这能解决当前痛点”。

正例：

| 主标题 | 副标题 |
|---|---|
| 拍了改、改了拍，识别不出来？ | 洋葱拍题精学，一拍识别，答案就是准 |

隐性反例：

| 主标题 | 副标题 | 问题 |
|---|---|---|
| 大题卡壳 | 洋葱看点，拿下满分 | 像标签云，主谓不完整；“拿下满分”有合规风险 |

## 双图

双图 = `短句1 + 短句2`。两句必须有内在关系，不是两个孤立卖点。

| 字段 | 作用 | 建议 |
|---|---|---|
| 短句1 | 抛痛点、场景或反差 | 6-12 字 |
| 短句2 | 给解法、动作或结果 | 12-18 字 |
| 短句3 | 不填 | 双图不用 |

常见关系：

- 痛点 -> 解法。
- 场景 -> 动作。
- 旧方式 -> 新方式。
- 问题 -> 结果。

应用商店双/三图要额外检查：图 2 或图 3 至少有一句明确出现“洋葱/洋葱学园”或具体产品动作，除非整组短句是连续的“你->你->你”命令式表达且画面中品牌足够明确。

正例：

| 短句1 | 短句2 | 关系 |
|---|---|---|
| 作业憋半天？ | 拍题精学找卡点，高效掌握不熬夜 | 痛点 -> 解法 |

隐性反例：

| 短句1 | 短句2 | 问题 |
|---|---|---|
| 解析秒出 | 专属老师陪你学 | 两句都像卖点标签，缺少关系和场景 |

## 三图

三图 = `短句1 + 短句2 + 短句3`。三句要有节奏，能让用户顺着看完。

| 字段 | 作用 | 建议 |
|---|---|---|
| 短句1 | 起点：痛点、动作或场景 | 4-8 字 |
| 短句2 | 展开：解法、过程或第二卖点 | 4-10 字 |
| 短句3 | 收束：结果或记忆点 | 4-10 字 |

### 三图关系库

| 关系 | 适合场景 | 示例 |
|---|---|---|
| 痛点 → 解法 → 奇效 | 最通用，适合讲清楚“为什么需要它” | 复习没方向 / 洋葱定制课表 / 每天只学重点 |
| 动作 → 过程 → 结果 | 适合展示产品怎么一步步帮用户 | 先测薄弱点 / 再排冲刺课 / 学完配套练 |
| 排比并列 | 适合做强节奏、强记忆点 | 不乱翻资料 / 不硬刷套卷 / 不浪费冲刺时间 |
| 旧方式 → 新方式 → 反差结果 | 适合做对比和惊艳感 | 以前凭感觉刷 / 现在按课表学 / 半小时也有重点 |
| 场景 1 → 场景 2 → 场景 3 | 适合覆盖多个使用瞬间 | 晚上没时间 / 周末补漏洞 / 考前抓重点 |
| 功能 1 → 功能 2 → 总结 | 适合产品能力较多时收束 | 诊断薄弱点 / 定制学习路 / 洋葱陪你冲期末 |
| 问题 → 问题 → 统一解法 | 适合前两图连续戳痛点，第三图给答案 | 课太多看不完 / 题太多刷不动 / 洋葱帮你挑重点 |
| 递进排比 | 适合兼顾节奏和逻辑 | 先学高频考点 / 再练对应题型 / 最后复盘薄弱点 |

`痛点 → 解法 → 奇效` 只是三图关系库的一类，不是默认唯一结构。先看功能、用户和渠道，再选关系。

期末冲刺课表、定制课表、私教班备考计划这类方向，优先考虑三类：`痛点 → 解法 → 奇效`、`旧方式 → 新方式 → 反差结果`、`递进排比`。这三类既适合三图，也最能把“不是让你更努力，而是帮你学得更准”讲出来。

正例：

| 短句1 | 短句2 | 短句3 | 关系 |
|---|---|---|---|
| 题目卡壳一拍精学 | 归纳重点解锁大招 | 智能推题举一反三 | 动作 -> 过程 -> 结果 |

隐性反例：

| 短句1 | 短句2 | 短句3 | 问题 |
|---|---|---|---|
| 拍题精学 | 分步解析 | 学习神器 | 三句都是名词标签，缺少叙事推进 |

## 多套生成

同一方向下生成多套时，不要让所有文案共享同一个句式。

可变化的维度：

- 切入角度：钩子 / 共情 / 数字证据 / 反差 / 场景。
- 场景细节：晚 9 点作业 / 周末刷题 / 小考订正。
- 用户视角：默认学生视角；用户明确指定家长/老师/其他人群时才覆盖。
- 图间关系：痛点解法 / 动作结果 / 旧新对比 / 多场景平铺 / 递进排比 / 问题问题统一解法。

如果用户要求“仿这个风格”，保留样本的节奏和字段结构，但至少换一个核心要素：卖点、场景、人群或结果。
