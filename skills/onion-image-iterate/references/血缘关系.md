# 血缘关系

`image_groups` 表的 `父图组` 字段用于记录新图组基于哪套旧图组产生。

## 字段语义

| 场景 | 父图组 |
|---|---|
| `onion-image` 原创新图 | 留空 |
| `onion-image-iterate` 基于 G-XXX 扩同类 / 微调 / 换形式 | 填原图组 record_id |
| 用户上传图片且不来自 Base 图组 | 留空 |
| 用户明确要求不标血缘 | 留空，但要在回复中说明会影响追溯 |

`父图组` 是 link 到 image_groups 自身的字段。写入值必须是 record_id 数组，例如 `["recABC123"]`，不是用户看到的 `G-005`。

## 如何取得 record_id

用户给 `G-005` 时，需要先读取 image_groups 中对应记录，保存返回的 `record_id`。这个 `record_id` 一路传给 `shared/scripts/write_image_group.py` 的 `--parent-group-id`。

不要把 `G-005` 字符串直接写入父图组字段。

## 写入工具

使用 `shared/scripts/write_image_group.py` 写新图组：

- `--parent-group-id recABC123`：有血缘。
- `--parent-group-id null`：原创或用户明确不标血缘。

脚本会把父图组转换为 Base link 字段需要的数组值，并负责图1/图2/图3附件上传。

## 复盘价值

血缘能回答：

- 这套新图是从哪套图扩出来的。
- 某个爆款图被扩了多少次。
- 扩同类、微调、换形式后续哪类更值得投入。

当前 MVP 没有跑量字段，因此只负责记录血缘，不做效果分析。

## 决策检查表（内部）

Use this checklist privately. Only summarize issues to the user if asked why.

- 基于 Base 旧图组迭代时，父图组是否填了原图组 record_id。
- 是否误把 G-XXX 当 record_id。
- 同批多个新图组是否都指向同一个原图组。
- 用户上传外部图片时，是否明确这是原创还是要关联某个 Base 图组。
