# 渠道与版位索引

> 这是路由索引。完整结构化规则在 `../config/channel-placement-rules.json`，来源是 2026-05-18 的 `research/渠道版位规则汇总.md`。

## 什么时候读哪份

| 当前问题 | 读取 |
|---|---|
| 用户给了渠道但没给版位 | `版位比例与尺寸.md` 和配置页 |
| 需要确定 `render.py --size` | `../config/channel-placement-rules.json` 的 `render_size` |
| 需要导出目标尺寸和 KB | `../config/channel-placement-rules.json` 的 `target_size` / `max_file_size_kb` |
| 同一文案要出多个渠道/版位 | 配置页多选版位；每个版位单独 prompt |
| 要写渠道化 prompt 风格 | `渠道风格与复盘.md` |
| 想参考 4-5 月跑量复盘、学习机风格 | `渠道风格与复盘.md` |

## 总原则

- 用户只选版位，不单独选比例。比例、gpt 出图尺寸、目标尺寸和压缩 KB 都由版位规则绑定。
- 同一个文案可以一次选择多个渠道和多个版位，例如 OPPO 横版大图 + 网易信息流通用。它们共用业务配置，但每个版位必须单独生成 prompt。
- 随机 IP、随机字体是逐图随机，不因为多版位共用配置而锁定同一个资产。
- 配置页必须展示 `target_size`、`render_size`、KB、直出能力和置灰原因。
- 一期开放 `direct` / `near` 且 `enabled=true` 的单图、双图、三图版位；配置页可按本次文案形式临时过滤不匹配版位。
- 扩图、合成、分层模板类版位先置灰；这些需要额外后处理，不由 `onion-image` 直接生成。
- 写入 Base 前默认导出为 JPG：先按 `target_size` 做 cover resize / center crop，再按版位 KB 压缩；未知 KB 默认 200KB。
- 各家渠道规则会变，实际投放前以投放后台当前规格为准；本 reference 是生成和入库阶段的工作基准。
