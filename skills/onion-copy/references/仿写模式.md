# 模式 A vs B 判断（出文案 skill）

> 启动时第一件事就是判断走 A 还是 B。

---

## 模式 A：新建生成（默认，严格流程）

**触发条件**（任一）：
- 用户指定方向 ID + 渠道/形式（"用 D-007 出信息流文案 3 套"）
- 用户没说什么（"出文案"）→ 反问方向 / 渠道 / 形式
- 用户粘了一条方向卡（不是文案）→ 用粘的方向出新文案

**特点**：
- 必填项严格检查（方向 / 渠道 / 形式 缺 → 反问，给具体选项）
- 走完整字段填写流程
- 用户敲定后**默认写入 Base**
- 写完同步把上游 directions 表 status 改"已用"

---

## 模式 B：扩文案（宽松流程）

**触发条件**（必须同时满足）：
1. 用户**粘了一条/几条完整文案样本**（含主+副 或 短句 1/2/3）
2. 用户用了仿写关键词：
   - "类似的再来 N 套"
   - "换个说法再写"
   - "这种风格的"
   - "仿照这条文案"
   - "同款再来"

**特点**：
- 不反问必填项（看样本自己推断渠道/形式/关联方向）
- 仿样本风格出 N 套
- 出完才问"要不要入库"（默认偏向不存）

---

## 模式 B 详细执行

### B1：看样本自己推断

| 要推断的项 | 怎么从样本里看 |
|---|---|
| **渠道 / 形式** | 字段结构（有主+副 → 单图；有短句 1/2 → 双图；有短句 1/2/3 → 三图）|
| **关联方向** | 用户没说 → 暂时留空；用户提到了某 D-XXX → 关联上 |
| **数量** | 用户说几套就几套；没说 → 默认 2 套 |

### B2：直接出 N 套

仿样本句式 / 结构 / 字数风格，但**内容是新角度**（避免与原样本同义复述）。

### B3：出完弹卡问入库

```
┌────────────────────────────────────┐
│ N 套文案已出。要存进 Base 吗？     │
│  ⚪ 存（追加到 copies）             │
│  ⚪ 不存（仅供参考）                │
└────────────────────────────────────┘
```

如果选"存" → 按主 SKILL.md 的工具调用规则写入 `copies`；如果有关联方向，同步把上游 `directions.status` 改为"已用"。

---

## 决策树

```
用户输入 →
  ├─ 粘了文案样本 + 用了仿写关键词？
  │   ├─ 是 → 模式 B（扩文案）
  │   └─ 否 → 模式 A
  └─ 模式 A 内部：
      ├─ 给了方向 + 渠道 + 形式 → 直接出
      ├─ 给了方向但缺渠道/形式 → 反问
      └─ 啥都没给 → 反问拉 Base 最近 5 条方向让用户选
```

---

## 跟 onion-image-iterate 扩/换图的区别

⚠️ **不要混淆**：

- **onion-copy 的模式 B = 扩文案**：用户给文案样本，要新文案
- **onion-image-iterate = 扩图**：用户给图组样本，要新图

如果用户给的是图组 ID 或图本身，应该让 `onion-image-iterate` 处理，不是 `onion-copy`。
