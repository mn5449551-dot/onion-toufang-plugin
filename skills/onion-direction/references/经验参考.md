# 经验参考：误区 + 复盘洞察

> 出方向时 AI 自查必看。**误区** = 不要怎么做；**洞察** = 怎样做更可能跑得好。

---

> 出方向时 AI 自查必看。命中任一 → 自己重写不要给用户。

---

## 误区 1：同义复述

### 表现
N 条方向看似不同，本质是"快、好、对、准、稳"的不同说法。

### 反例
```
方向 A：拍题精学，秒出答案
方向 B：拍题精学，快速精准
方向 C：拍题精学，效率提升
```
→ 都在说"快"，没真正不同的卖点。

### 检查方法
单独抽 N 条的「字段 6 奇效」字段排成一列，**能一眼分辨吗**？分不清 → 至少 1 条要重写换卖点角度。

### 怎么避免
出方向前先想清楚 N 个角度：
- 「快/慢」是一个角度（效率类）
- 「懂/不懂」是一个角度（精学类）
- 「单独定位/泛泛」是一个角度（精准类）
- 「人物/工具」是一个角度（情感类）
→ N 条至少各选不同角度。

## 误区 2：痛点不够痛

### 表现
"作业难"、"学习压力大"、"用户需要高效学习" —— 常态描述不是痛点，是行业空话。

### 反例
```
字段 4：学生作业很多，需要快速做题
```
→ 没场景没细节没情绪，谁看了都没感觉。

### 检查方法
**能拍成 3 秒短视频吗**？拍不出 → 没画面感 → 重写。

### 怎么避免
- 时间要具体（"晚上 9 点"、"周日下午"）
- 行为要具体（"五道大题"、"卡在第三步"）
- 情绪要具体（"越等越烦躁"、"只想抄答案"）
- 至少 2 个具体细节才合格

---

## 误区 3：解法滑向功能清单

### 表现
解法字段写成产品说明书，列功能而不是讲使用体验。

### 反例
```
字段 5：洋葱拍题精学有 AI 错题本、分步解析、个性化学习路径...
```
→ 是说明书，不是营销文案。

### 检查方法
解法里**有反差或比喻吗**？没有 → 重写。

### 怎么避免
- "不是 X，而是 Y" 句式（反差）
- "像 X 一样 Y" 句式（比喻）
- 描述**使用感受**，不是列功能列表

### 正确示例（真实案例提炼）
```
字段 5（识别准确）：答案经过 AI + 名师教研团队校准——不是 AI 瞎编的“看着像对”，是真的对。
```
反差和可信机制都有。

## 误区 4：奇效宽泛

### 表现
"高效"、"提分"、"省心" —— 谁都能写、记不住、没区分度。

### 反例
```
字段 6：用了洋葱拍题精学，学习更高效
```
→ "高效" 是空话。

### 检查方法
奇效**必须呼应字段 4 同一场景的具体改变**。比如字段 4 写"晚上 9 点做作业等加载"，字段 6 就必须写"现在拍完即看，不用等"。两个字段没关联 → 重写。

### 怎么避免
- 用"以前 / 现在" 对比句式
- 具体到时间 / 动作的改变（"以前 10 分钟，现在 30 秒"）
- 结尾给"获得感总结"（"省的是专注力" / "做题效率翻倍"）

## 误区 5：人群描述模糊

### 表现
"K12 学生"、"中考家长"、"初中生" —— 太宽，没特征。

### 反例
```
目标人群：初中生
```
→ 任何方向都能套，没意义。

### 检查方法
人群描述**含行为/情境特征吗**？没有 → 重写。

### 怎么避免
- 加行为特征：「时间紧」「光看答案跟不上」「不爱问家长」
- 加场景：「周中晚间赶作业」「测验后订正」「周末集中性写作业」
- 30-50 字一段（参考即可，合理超出 OK）

### 正确示例
```
目标人群：时间紧、作业量大、对"等待"零容忍的学生（尤其周中晚间赶作业群体）
```
有行为（时间紧、作业量大）、有特征（零容忍）、有场景（周中晚间）。

## 自查口诀

每次出完 N 条方向，按这个顺序扫一遍：

1. **同义复述**：N 条「奇效」并排能区分吗？
2. **痛点强度**：每条「场景问题」能拍 3 秒短视频吗？
3. **解法反差**：每条「解法」有反差/比喻吗？还是在列功能？
4. **奇效呼应**：每条「奇效」跟「场景问题」是同一场景的前后对比吗？
5. **人群具体**：每条「目标人群」含行为/场景特征吗？

5 个都过 → 给用户看；任一不过 → 自己改完再给。

## 渠道偏好洞察

| 渠道 | 偏好类型 | 原因猜测 |
|---|---|---|
| **学习机** | 学生视角（NPC 老师等）| 学习机用户是学生本人，情感共鸣强 |
| **应用商店** | 对比型（识别准确 vs 别家）| 商店是主动搜索场景，对比转化高 |
| **信息流** | 时间敏感型（考前 / 周末）| 信息流是被动看到，时间锚点 + 紧迫感容易抓住注意力 |

## 高跑量方向的共性

1. **具体场景型** > 通用型
   - "周末赶作业" 优于 "学习压力大"
   - "考前两周" 优于 "想提分"

2. **时间紧迫感** 容易转化
   - "晚上 9 点" / "考前" / "周末集中" 这类时间锚点很有力

3. **学生视角** 在学习机和小米渠道表现好
   - 不只是家长视角

4. **验证性卖点**（让用户验证 vs 别家）跑得稳
   - "拍三遍 vs 一拍精准"
   - "看不懂 vs 讲到会"

---

## AI 出方向时怎么用这些洞察

1. **不硬套**：不要每条都模仿"识别准确"的写法；多样性比模仿冠军更重要
2. **可参考**：N 条方向里**至少 1 条**包含一个"已验证好用"的角度（时间紧迫感 / 学生视角 / 验证性对比 等）
3. **告知用户**：如果用户问"哪条更可能跑"，可以根据这份洞察给建议，但说明是"基于 4-5 月历史数据"，不保证未来一定行
