# 迭代力度与改动轴

力度决定改动幅度，轴决定改哪里。先定力度，再定轴；两者不要混成一套固定流程。

## 力度

| 力度 | 保留程度 | 可以改 | 不该改 |
|---|---|---|---|
| 微调 | 约 95% | CTA、Logo、比例、局部细节、轻微风格 | 图片形式、方向、核心卖点、大场景 |
| 扩同类 | 约 70% | 文案、IP、场景细节、构图、部分风格 | 方向、核心卖点、图片形式 |
| 换形式 | 推翻重做 | 方向、文案、图片形式、视觉叙事 | 血缘记录不能丢 |

## 可解锁轴

| 轴 | 说明 | 可用于 |
|---|---|---|
| IP | 豆包换上官等角色替换 | 微调、扩同类 |
| 文案 | 同方向下换一套新文案 | 扩同类 |
| 场景细节 | 书桌换教室、晚上换周末下午 | 扩同类 |
| 比例 | 1:1 改 3:2 或 16:9 | 微调、扩同类 |
| Logo / CTA | 换 Logo 版本或按钮文字 | 微调 |
| 风格参考图 | 同内容换风格，但保留主体关系 | 微调、扩同类 |
| 图片形式 | 单/双/三图互换 | 换形式 |
| 方向 | 换上游心智 | 换形式 |

## 判断边界

- 只换 CTA：微调。
- 只换 IP，其他全部保持：微调 x IP。
- 换 IP + 场景 + 文案：扩同类。
- 单图改三图：换形式。
- 换方向或卖点：换形式。
- 用户说“跑得好，再来 3 套类似”：扩同类。
- 用户说“这套不出量，换思路”：换形式。

## 决策检查表（内部）

Use this checklist privately. Only summarize issues to the user if asked why.

- 当前力度是否匹配用户真实改动幅度。
- 用户指定的轴是否都被覆盖，是否额外加入了没要求的变化。
- 微调是否真的只改 1-2 个细节。
- 扩同类是否保持方向、卖点和图片形式。
- 换形式是否保留 `parent_group_id` 血缘。
- 新图是否仍符合原渠道和合规要求。
