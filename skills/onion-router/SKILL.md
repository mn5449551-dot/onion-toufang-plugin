# onion-router

## 任务目标

这是洋葱投放插件的总分诊台，只负责分流，不生成方向、不写文案、不生图、不写 Base。它的输出应是：当前请求该进入哪个 skill、是否缺阻塞信息、下一步要问什么或调用什么。

不要把 router 当业务执行器。分流后立刻交给对应 skill 的规则，不在 router 内复述完整业务流程。

## 快速分流

| 用户信号 | 目标 skill | 处理 |
|---|---|---|
| 环境检查、配置好了吗、Base 结构、pending、API key、图没入库、图片不显示 | `onion-help` | 进入诊断，不跑业务生产 |
| 出方向、方向卡、素材方向、D-XXX 扩方向/改方向 | `onion-direction` | 方向生成、扩写、改版、新版入库 |
| 出文案、写文案、选择方向出文案、D-XXX 出文案、C-XXX 扩文案/改文案 | `onion-copy` | 方向到文案，或基于已有文案扩写/改版 |
| 用文案出图、C-XXX 出图、临时文案出图、刚选的文案出图 | `onion-image` | 从文案生成全新图片 |
| G-XXX 扩同类/微调/换形式，旧广告图同款再来，上传旧图换文案/换角色/换 CTA | `onion-image-iterate` | 基于已有图或图组迭代 |

## ID 分流

- `D-XXX` 是方向锚点：可以进入 `onion-direction` 扩/改方向，或进入 `onion-copy` 出文案；D-XXX 不能直接出图。用户说“D-XXX 出图”时，先转 `onion-copy` 生成或确认文案。
- `C-XXX` 是文案锚点：用户说扩、改、换说法时进 `onion-copy`；用户说出图、生图、做图时进 `onion-image`。
- `G-XXX` 是图组锚点：用户说扩、换、微调、同款再来时进 `onion-image-iterate`。
- 无 ID 但有临时文案：用户要出图时进 `onion-image`，不强制补方向或文案 ID。

## 选择语义

- 选择方向：用户说“选择方向 2 / 方向一 / 就用这个方向”时，交给 `onion-direction`。方向入库后必须问下一步：出文案、继续改方向，还是先停。
- 选择文案：用户说“选择第 2 条 / 我选这个文案”但没说下一步时，交给 `onion-copy` 问清是入库、基于它出图，还是继续改文案；不要擅自出图。
- 选择图片：用户打开选择页后，不接受“选 set1 / 选第 2 套”这类口头选择 set 来入库。必须回到 `onion-image` 或 `onion-image-iterate`，要求用户在选择页标注并提交，或粘贴同结构 JSON。
- 多选：选择多个方向或多个文案后要求下游生产时，必须问每个都继续，还是只基于其中一个继续；不要默认第一个。

## 上传图先判断角色

上传图先判断角色，不要因为“用户上传了图”就默认进 `onion-image-iterate`。

| 上传图角色 | 路由 |
|---|---|
| 旧广告图锚点：照这张改、同款、扩同类、换文案、换角色、换 CTA | `onion-image-iterate` |
| APP 截图、功能界面截图、拍题结果页截图 | 当前新图任务的参考素材，通常仍走 `onion-image` |
| 风格参考图：色调、构图、质感参考 | 当前新图任务的参考素材，通常仍走 `onion-image` |
| IP 参考图：新增角色或临时角色形象 | 当前新图任务的参考素材，通常仍走 `onion-image` |
| 用户只上传图但没说用途 | 先问：“这张图是作为旧广告图来改，还是作为 APP 截图/风格/IP 参考图？” |

## 反问原则

只问会影响分流的阻塞项：

- 不知道用户要方向、文案、新图、改旧图还是环境检查时，问一句让用户选目标。
- 上传图用途不明时，只问图片角色。
- 用户只说“选择第 N 条”且上下文不足时，问选择的是方向、文案还是图片方案。
- 用户想直接用 D-XXX 出图时，不问无关配置，直接说明需要先形成文案并转 `onion-copy`。

一旦能分流，不要继续在 router 里追问业务细节；交给目标 skill。

## 硬约束

- router 不能写 Base，不能调用 `render.py`，不能启动 `/image-config`，不能生成候选方向/文案/prompt。
- D-XXX 不能直接进入 `onion-image`。
- 口头选择 set 不能触发图片入库；图片入库依据必须是选择页结果 JSON。
- 上传图如果只是 APP 截图、风格参考图或 IP 参考图，不能误转 `onion-image-iterate`。
- 路由到目标 skill 后，必须遵守目标 skill 的入库、配置页、选择页、截图闸门和验证规则。
