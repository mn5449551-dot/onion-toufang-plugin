# 候选回复格式

用户可以自然表达，Agent 对外回复要稳定、好选、能直接进入下游。候选内容不是内部推理日志；不要把切入角度、评分、后续图片建议或排版判断展示给用户。

## 方向候选固定排版

每条方向必须完整展示 6 个方向字段。不能写省略号，不能写“同上”，不能只给摘要，不能把字段 4/5/6 合并成一句。

```text
方向 1｜<素材方向完整标题>
功能：<功能选项>
卖点：<1-3 个卖点>
目标人群：<行为和情境特征>
适配阶段：<阶段>（<用户指定 / 按当前日期推断 / 默认日常学习>）

1 能解决用户在“具体哪个场景里的哪个问题”：
<完整场景问题>

2 能带来什么不一样的“一听很惊艳”的解法？
<完整差异化解法>

3 因此带来了哪个场景下的什么“奇效”？
<完整可感知变化>
```

方向候选只展示业务字段和必要的阶段来源说明。内部切入维度、多样性判断、质量自检和后续图片建议留在内部。

## 文案候选固定排版

单图：

```text
文案 1｜<渠道> · 单图
主标题：<主标题>
副标题：<副标题>
关联方向：<D-XXX / 临时方向 / 无>
```

双图：

```text
文案 1｜<渠道> · 双图
短句1：<图1文字>
短句2：<图2文字>
关联方向：<D-XXX / 临时方向 / 无>
```

三图：

```text
文案 1｜<渠道> · 三图
短句1：<图1文字>
短句2：<图2文字>
短句3：<图3文字>
关联方向：<D-XXX / 临时方向 / 无>
```

`文案类型` 只用于内部多样性和必要 Base 字段；候选回复不要展示。多条文案必须逐条编号，字段名保持一致，方便用户直接说“选第 N 条”。
