# 视觉元素规范（IP 形象 / 风格参考图 / 字体）

> 按需读取。涉及 IP 形象、Logo、字体、风格参考图或用户上传参考图时读对应章节；各视觉元素都存本地 `assets/`，Base 字段 text 存相对路径。**用户上传新风格图走 attachment 字段，等维护人定期整理升级到内置库**。

---

> 视觉配置需要 IP 时，AI 必须从 `assets/asset-manifest.json` 读取本地真实存在的参考图清单让用户选。不要只硬编码“豆包 / 上官 / 小锤 / 雷婷 / 豆花 / 狗蛋”，因为 manifest 里可能有学段、全身照、Q3 等变体。

## 本地参考图组织

内置资产以 `assets/asset-manifest.json` 为准。新增资产必须按 `references/资产命名与参考图标注.md` 的标准命名。

当前 IP 标准路径按角色聚合，不按学段拆文件夹：

> ⏸️ **MVP 启动期**：本地参考图库可能不全。视觉配置时只列**本地真实存在**的 IP 子目录和图，不要列空的。

## 选完 IP 后，AI 怎么读参考图

```
1. 用户在视觉配置卡选了"豆包"
2. AI 从 `assets/asset-manifest.json` 找到 `ip.doubao.junior.standard.001`
3. 未指定学段/变体时优先用 `standard`；用户指定全身照、学段或 Q3 时，改用 manifest 里的 `fullbody` / `qstyle-q3` 精确资产
4. AI 在 Base 字段 `IP参考图引用` 填 manifest 的标准 `path`
5. 调 render.py 时用 `reference_images` 对象传 `label`、`role`、`asset_id`、`path`
```

## 写进 prompt 怎么处理

prompt 里**显式提及 IP 名 + 版本**，并通过 `references` 字段传参考图：

```json
{
  "prompt": "参考图说明：参考图1 是品牌 Logo；参考图2 是豆包正常版角色参考图。竖版 9:16 信息流广告图，左上角使用参考图1的 Logo；参考图2的豆包站在屏幕旁，保持参考图2的脸型、发型、服装和活泼亲和感，手举一道数学大题……",
  "reference_images": [
    {"label": "参考图1", "role": "品牌 Logo 参考图", "asset_id": "logo.onion.standard.001", "path": "assets/logos/onion-logo-standard-001.png"},
    {"label": "参考图2", "role": "豆包正常版角色参考图", "asset_id": "ip.doubao.junior.standard.001", "path": "assets/ip-roles/doubao/doubao-junior-standard-001.png"}
  ]
}
```

## 参考图顺序约定

生成 base 图时，参考图按稳定顺序传，prompt 里也按这个顺序解释：

1. Logo（如有）→ `参考图1`
2. IP 角色（如有）→ `参考图2`
3. 字体参考（如启用洋葱专属字体）
4. 风格参考图（如有）
5. 用户额外参考图

如果某项没有使用，后面的编号前移；不要空出 `参考图2`。

branch 图默认只传 base PNG，不重复传 Logo/IP/字体/风格，除非用户明确要换其中某个资产。

## 多 IP 配合规则

| 场景 | 建议 |
|---|---|
| 同一套图（双/三图）| **同一个 IP 形象一致**（不要图 1 用豆包、图 2 换上官 — 系列感断）|
| 不同套图之间 | 可以换 IP（增加多样性）|
| 用户明确要"换 IP 试" | 按用户要求，但同套保持一致 |

## 洋葱专属字体默认规则

老张 API / gpt-image-2 渲染中文字体时，默认使用洋葱专属字体参考图，而不是通用系统字体。

默认规则：

- 用户未指定字体时，不追问字体选择；从 `assets/font-references/` 的 31 张字体参考图里随机/轮换选 1 张。
- 同一套双/三图使用同一张字体参考，保持系列一致；不同套可以换不同字体参考，增加多样性。
- 字体参考是“风格参考”，不是精确复刻。prompt 要写：学习参考图的字形气质、描边、微倾斜、标题排版节奏，让文字和当前画面融合；不要 100% 照抄，不复制参考图里的示例文字、颜色或背景。
- 用户明确说“参考 03 / 字体参考 03 / 用这张字体”时，才用指定编号。
- 除非用户明确要求，否则不要切到 OPPO Sans、HarmonyOS Sans、思源黑体、阿里巴巴普惠体等通用字体。

```
参考图说明：参考图1 是豆包角色参考图；参考图2 是洋葱专属字体参考图。
画面文字「拍了改、改了拍，识别不出来？」学习参考图2的字形气质、描边和标题排版节奏，
但要和当前画面的配色、光线、构图融合；不要求与参考图2完全一致，不复制参考图2中的示例文字。
```

## 洋葱专属字体参考

`assets/font-references/` 目前有 31 张洋葱广告文字风格参考图。默认就启用洋葱专属字体参考：

- 从 31 张里随机/轮换选 1 张作为字体参考图。
- prompt 只让模型学习字形气质、描边、微倾斜和挂件装饰，不照抄参考图颜色。
- 同一套双/三图必须用同一张字体参考，branch 通过 base PNG 继承字体，不重复传。

## ⚠️ AI 渲染中文文字的限制

GPT-Image-2 渲染中文有时不稳定（特别是复杂笔画 / 大字号）。如果发现：
- 文字模糊 / 错字 / 笔画错乱 → prompt 里加"清晰锐利的中文字渲染，字形完整"
- 或者降低文字数量（用更短的标题）
- 极端情况下让 AI 留出文字位置（白色矩形占位），后期 Pillow 加文字
