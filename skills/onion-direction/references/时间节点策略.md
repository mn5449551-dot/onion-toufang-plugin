# 时间节点策略

本文件用于 `onion-direction` 在用户提到日期、节点、假期、考试或季节时选择方向场景。它不是时间表操作手册，而是节点语义判断规则。

业务知识源：

- `business-knowledge/canonical/时间节点推荐表-2026-通用.md`
- `business-knowledge/references/for-stage-2b/时间节点/寒暑假.md`
- `business-knowledge/references/for-stage-2b/时间节点/开学季.md`
- `business-knowledge/references/for-stage-2b/时间节点/期中期末.md`

## 核心原则

时间节点不是一个标签，而是用户此刻最可能遇到的学习状态。

生成方向时，先用时间节点推断：

- 学生当前任务：预习、收心、复盘、备考、查漏补缺、持续学习。
- 家长/学生的真实担忧：跟不上、不会规划、复习没重点、假期松懈、作业卡壳。
- 更适合调用的产品能力：同步课、拍题精学、错题本、精准复习、总复习、学情报告。

不要只把“开学季/期末前/暑期”写进 `适配阶段` 就结束；它必须影响字段 4/5/6 的场景问题、解法和可感知变化。

## Base 阶段映射

当前 Base 的 `适配阶段` 不是完整 `timeNode` 六枚举，只能写已有选项：

| 业务 timeNode | Base `适配阶段` |
|---|---|
| `daily` | 日常学习 |
| `winter` | P0-寒假 |
| `summer` | 暑期 |
| `back_to_school` | P0-开学 |
| `midterm` | 期中前 |
| `final` | 期末前 |
| 无法归并 / 城市例外 / 用户自定义节点 | 其他 |

细分中文标签不写入 Base 字段，但要进入方向语义，例如“期中复盘与五一后调整”“暑假后期与秋季预习”“春季开学节奏适应”。

## 节点策略

### 开学季 / back_to_school

核心状态：收心、适应新节奏、新知识预习、学段跃迁。

高价值场景：

- 暑假或寒假后，孩子玩心还在，第一周进入不了学习状态。
- 小升初、7 升 8、8 升 9，难度、科目和节奏突然变化。
- 新学期第一批作业不会做，课上没听懂，回家没人能及时讲。

适合卖点：

- 同步课、动画视频课、5-8 分钟知识点、教材同步。
- 拍题精学、步骤拆解、哪步不懂点哪步。
- 学情诊断、学习规划、错题本。

### 期中 / 期末 / final / midterm

核心状态：复习规划、查漏补缺、重点难点、高频错题、考后复盘。

高价值场景：

- 临近考试，孩子面对多个科目不知道先复习什么。
- 复习只会翻书，抓不到重点、难点、易错点。
- 课上听懂，一做题就不会，暴露出一知半解。
- 考后不知道怎么总结错题，下一次还在同一类题上卡住。

适合卖点：

- 总复习、精准复习、试卷库、考前突击。
- 错题本、高频错题、得分点拆解。
- 同步知识点搜索、动画课查漏补缺。

合规边界：

- 不写“保过”“必涨”“提分 N 分”。
- 用“复习有重点”“错题可复盘”“一类题能讲透”等行为和能力变化替代结果承诺。

### 寒暑假 / winter / summer

核心状态：假期松懈、不会规划、复习旧知识、预习新知识、弯道准备。

高价值场景：

- 放假后学习状态断掉，开学前才发现跟不上。
- 期末结果出来后不知道怎么补，假期计划空泛。
- 家长想让孩子提前学，但不知道从哪一课开始。
- 补习班学习没重点，孩子不爱学，效率低。

适合卖点：

- 同步课、动画视频课、学练一体。
- 错题本、学情诊断、精准复习。
- 预习新知、短时高频、孩子可自主安排。

## 输出自查

生成方向前私下检查：

- 方向是否真的体现了时间节点里的学生状态，而不是只换了 `适配阶段`。
- 字段 4 是否有节点特有场景，例如“开学第一周”“期末前两周”“暑假后期”。
- 字段 5 的解法是否匹配节点任务，例如备考用总复习/错题本，开学用同步课/拍题精学。
- 字段 6 是否写行为变化，不写分数承诺。
- 如果使用了日期推断，输出时是否简短说明这是按当前日期或用户投放时间推断。
