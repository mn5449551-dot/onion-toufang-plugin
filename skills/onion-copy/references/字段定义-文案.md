# 文案字段定义

> 写每套文案时严格按这份字段规则。字段名用原版（不简化）。
> **字数都是建议范围**，少了多了都 OK——唯一硬标准是**句子通顺自然**。

---

## 字段总览（按渠道 × 形式组合）

| 渠道 | 形式 | 必填字段 |
|---|---|---|
| 信息流 | 单图 | 主标题 + 副标题 |
| 信息流 | 双图 | 短句1 + 短句2 |
| 信息流 | 三图 | 短句1 + 短句2 + 短句3 |
| 应用商店 / 学习机 | 单图 | 主标题 + 副标题 |
| 应用商店 | 双图 | 短句1 + 短句2 |
| 应用商店 | 三图 | 短句1 + 短句2 + 短句3 |

> ⚠️ **不在该形式范围的字段直接不传**（写 Base 记录时不要传空字符串，省略字段即可）
> 学习机只支持单图；应用商店 / 学习机单图共用文案逻辑。

## 字段详细规则

### 主标题（单图用）

字数建议 8-14 字（合理超出 OK）。

**规则**：
- ✅ **疑问 / 感叹** 起手（"拍了改、改了拍，识别不出来？" / "看完答案解析更懵了？"）
- ✅ 有**画面感**或**情绪锚点**
- ✅ 默认学生视角，先让学生觉得“这就是我”
- ❌ 不要平铺直叙（"洋葱拍题精学很好用" — 没钩子）

### 副标题（单图用）

字数建议 15-22 字（合理超出 OK）。

**规则**：
- ✅ **补完解法**（不要重复主标题信息）
- ✅ 含**产品动作 + 使用感受**（"洋葱拍题精学，一拍识别，答案就是准"）
- ❌ 不要功能罗列

### 短句 1/2/3（双 / 三图用）

字数建议：
- 双图：图 1 建议 6-12 字、图 2 建议 12-18 字
- 三图：每图建议 4-8 字

**关系类型由 AI 自选**（不限种类）：

参 `规格.md` 里的双图 / 三图真实案例提炼启发。常见但**非穷举**的关系：
- 痛点 → 解法
- 场景 → 动作
- 反差 → 亮点（否定→肯定）
- 功能 1 + 功能 2 + 总结（三图）
- 多角度平铺（人物 / 过程 / 体验）
- 动作 → 特点 → 结果（递进）

→ AI 看方向卡的 3 个原版字段（场景问题/差异化解法/奇效）自己判断哪种关系最贴。
→ 多套之间应混搭不同关系类型，避免每套都一样。

**通用规则**：
- ✅ 多句必须有某种**内在关联**（不能各句独立无关）
- ✅ 同一句的字数大致相当（视觉平衡）
- ✅ 每句都要是自然中文，主谓宾或省略关系必须清楚；宁可多 2-4 个字，也不要砍成病句。
- ❌ 不要硬套某种固定模式

### 拍题精学“快查 + 深学”写法

当方向核心是“先快查，再按需深入问”时，文案要把两段逻辑写顺：

1. 先说明洋葱的快动作：`洋葱一拍，解析秒出` / `洋葱拍题精学，先出答案解析`
2. 再说明后续能力：`看不懂，再问洋葱 AI` / `卡在哪步，就继续追问`

好例子：

| 形式 | 文案 |
|---|---|
| 双图 | 图 1：洋葱一拍，解析秒出<br>图 2：看不懂，再问洋葱 AI |
| 双图 | 图 1：先看答案解析<br>图 2：卡在哪步，就继续追问 |
| 单图 | 主：拍题先看答案解析<br>副：洋葱一拍秒出，不懂还能继续问 AI |

坏例子：

| ❌ 文案 | 问题 | ✅ 改法 |
|---|---|---|
| 先看答案解析 / 洋葱不懂还能继续问 | “洋葱不懂”变成洋葱看不懂，主语错 | 洋葱一拍，解析秒出 / 看不懂，再问洋葱 AI |
| 拍题秒出解析 / 洋葱 AI 把步骤讲清楚 | 只有功能并列，少了“不只是快”的递进 | 洋葱一拍先出解析 / 卡在哪步，AI 接着讲 |
| 洋葱先给重点，再逐步讲 | “重点”太虚，不像具体产品动作 | 先看标准答案解析 / 不懂哪步，继续追问 |

## 文案类型（select 单选，可空）

| 类型 | 特征 | 例 |
|---|---|---|
| 钩子 | 疑问句 / 强吸引 | "拍了改、改了拍，识别不出来？" |
| 共情 | 描述用户处境 | "晚上 9 点作业还没写完？" |
| 数字 | 含具体数字 | "5 道题 1 分钟搞定" |
| 反差 | 对比类 | "拍三遍 vs 一拍精准" |
| 故事 | 场景叙事 | "考前最后两周，他这样翻盘..." |

AI 自选；模棱两可时留空即可。多套输出时不要全选同一类：3 套至少覆盖 3 类；5 套至少覆盖 3 类且同一类最多 2 套；8 套及以上至少覆盖 4 类。

### 渠道偏好

| 渠道 | 优先类型 | 少用类型 |
|---|---|---|
| 信息流 | 数字、钩子 | 故事、铺垫很长的共情 |
| 应用商店 / 学习机单图 | 数字、反差、学生视角共情、功能动作型钩子 | 家长共情、硬背书、过软故事 |
| 应用商店多图 | 排比、递进、动作过程型 | 三句孤立功能标签 |

来源参考：`business-knowledge/references/for-stage-3b/文案生成参考/copy-types.md`。这里吸收类型选择逻辑，不新增 Base 选项。

## 字段名注意（写 Base 时）

传给 `write_record.py` 的字段名是中文，**不要简化**：

- 双/三图：`短句1` / `短句2` / `短句3`（注意是中文数字格式 `短句1` 不是 `短句 1`，按 base_schema.md 校对）
