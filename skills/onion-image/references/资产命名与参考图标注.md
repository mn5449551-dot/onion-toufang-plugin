# 资产命名与参考图标注

当出图需要 Logo、IP、字体、风格图或用户上传图时，先建立“参考图清单”，再写 prompt。不要只说“用豆包参考图”或“参考上图”，否则模型容易把 Logo、IP、风格和字体互相混用。

## 资产 ID 和路径

内置资产以 `assets/asset-manifest.json` 为准。每个资产有三个层次：

| 字段 | 作用 | 示例 |
|---|---|---|
| `asset_id` | 稳定业务 ID，用于记录和追踪 | `ip.doubao.junior.standard.001` |
| `path` | render.py 真实传入的文件路径 | `assets/ip-roles/doubao/doubao-junior-standard-001.png` |
| `prompt_role` | 写进 prompt 的用途说明 | `豆包正常版角色参考图` |
| `aliases` | 用户可能说出的全名、短名或口语名 | `["王小锤", "小锤"]` |
| `asset_name_zh` | 给人看的中文资产名，尤其用于多学段/多变体 | `李狗蛋-小学全身照` |

命名规则：

```text
IP asset_id: ip.<slug>.<stage>.<variant>.<number>
IP path: assets/ip-roles/<slug>/<slug>-<stage>-<variant>-<number>.png

Teacher asset_id: teacher.<slug>.<variant>.<number>
Teacher path: assets/teachers/<slug>/<variant>/<slug>-teacher-<variant>-<number>.png

Logo asset_id: logo.<slug>.<variant>.<number>
Logo path: assets/logos/<slug>-logo-<variant>-<number>.png
```

常用 `variant`：

| variant | 用途 |
|---|---|
| `standard` | 当前正常人物角色形象 / 默认参考图 |
| `fullbody` | 干净全身立绘，适合用户明确要“全身照”“立绘”时使用 |
| `qstyle-q2` / `qstyle-q3` | 飞书人物图库里按 Q2/Q3 标注的 Q 版全身图 |

示例：

```text
ip.goudan.primary.fullbody.001
assets/ip-roles/goudan/goudan-primary-fullbody-001.png

ip.goudan.junior.qstyle-q3.001
assets/ip-roles/goudan/goudan-junior-qstyle-q3-001.png
```

历史路径如 `assets/ip-roles/初中/豆包.png` 或 `assets/ip-roles/junior/doubao/standard/...` 不再保留；新增和引用一律走 manifest 的角色聚合路径。

当前投放配置页统一从 `assets/ip-roles/` 和 `asset-manifest.json` 读取可选角色。业务明确作为投放 IP 使用的老师形象也放入 `ip-roles`，用 `stage=teacher` 区分，例如 `ip.zhangwuxian.teacher.fullbody.001`、`ip.wenxin.teacher.fullbody.001`、`ip.nina.teacher.fullbody.001`。

## IP 名称归一

用户可能说“王小锤”“李狗蛋儿”“田豆花”等全名，也可能说“小锤”“狗蛋”“豆花”等短名。处理方式：

1. 先在 `assets/asset-manifest.json` 里用 `display_name` 和 `aliases` 匹配。
2. Base 字段 `IP形象` 写标准 `display_name`，不要写别名。
3. prompt 的参考图说明可同时写标准名和别名，例如“参考图2 = ip.xiaochui.junior.standard.001，小锤（王小锤）正常版角色参考图”。
4. 查不到本地资产时，不要臆造 asset_id；让用户提供原图文件，或把名字放入候选清单等待入库。

## 参考图标签

每次调用 render 前，按传入顺序给参考图编号，并在 prompt 里显式解释：

```text
参考图1 = logo.onion.standard.001，品牌 Logo，只用于左上角标识，不要改造为按钮或角色。
参考图2 = ip.doubao.junior.standard.001，豆包正常版角色参考图，保持脸型、发型、服装和整体比例。
参考图3 = font.ref.003，字体风格参考，只学习字重和排版节奏，不复制示例文字。
```

prompt 后续描述必须使用这些标签，例如“让参考图2的豆包站在手机屏幕右侧”“左上角使用参考图1的 Logo”。不要写“第一个图大概是 Logo”“上面的角色”“这个参考图”等含糊说法。

## render.py 输入格式

推荐用 `input-json` 的对象格式传参考图：

```json
{
  "prompt": "参考图说明：参考图1是品牌 Logo；参考图2是豆包正常版角色。按所选版位的构图做广告图，左上角使用参考图1的 Logo，参考图2的豆包站在屏幕旁指向解析卡片……",
  "size": "1024x1664",
  "quality": "medium",
  "reference_images": [
    {
      "label": "参考图1",
      "role": "品牌 Logo 参考图",
      "asset_id": "logo.onion.standard.001",
      "path": "assets/logos/onion-logo-standard-001.png"
    },
    {
      "label": "参考图2",
      "role": "豆包正常版角色参考图",
      "asset_id": "ip.doubao.junior.standard.001",
      "path": "assets/ip-roles/doubao/doubao-junior-standard-001.png"
    }
  ]
}
```

`render.py --validate-only` 会检查：

- 参考图路径存在；
- 显式传了 `label` 的参考图必须在 prompt 里出现；
- 标签不能重复；
- 输出 metadata 会回显 `reference_image_labels`，便于排查传图顺序。

## 单图 / base 图

单图和双/三图的图 1 使用完整参考图：

1. Logo
2. IP 角色
3. 字体参考
4. 风格参考图
5. 用户额外参考图

如果某类资产没用，就跳过，不保留空位。例如只用 Logo + IP，则 `参考图1=Logo`、`参考图2=IP`。

## branch 图

双/三图的图 2/3 默认只传图 1 PNG：

```text
参考图1 = set1_img1.png，同套图第 1 张基准图。保持参考图1的角色、Logo、字体、配色、构图和光线，只改变屏幕内容、角色动作和本张文字。
```

只有用户明确要换 IP、换 Logo、换风格时，branch 才额外传新资产，并重新解释参考图：

```text
参考图1 = set1_img1.png，基准图。
参考图2 = ip.shangguan.junior.standard.001，新角色上官正常版，只替换原图中的角色，其余保持参考图1。
```

## 写 Base

- `IP参考图引用` 写 manifest 中的 `path`，不是旧中文路径。
- `Logo参考图引用` 写 manifest 中的 `path`。
- `风格参考图引用` 写内置风格图 path；用户上传风格图写附件字段。
- 图组 prompt 字段里保留“参考图说明”段，方便以后复盘为什么图像对应错。

## 新增资产流程

1. 放到标准路径，文件名用 `<slug>-<stage>-<variant>-<number>.png`。
2. 在 `assets/asset-manifest.json` 增加一条资产记录。
3. 若是 IP 新变体，`variant` 用 `standard`、`fullbody`、`qstyle-q2`、`qstyle-q3`、`3d`、`pose` 等稳定枚举，不临时写“新版”“最终版”。
4. 新角色先确认官方名和用途边界；不确定名字时先用 `candidate-<number>` 暂存文件，不写入 manifest 的正式 IP 列表。
5. 跑 `python3 -m json.tool assets/asset-manifest.json` 和 `render.py --validate-only` 示例，确认路径能解析。
