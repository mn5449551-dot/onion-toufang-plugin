# 模式 A vs B 判断 + 模式 B 执行流程

> 出方向 skill 启动时第一件事就是判断走 A 还是 B。判断逻辑越快越好，别拖。

---

## 模式判断

### 模式 A：新建生成（默认，走严格流程）

**触发条件**（任一）：
- 用户从零开始（说想做某个功能 / 某个卖点的方向）
- 用户给了功能 + 卖点 + 数量等参数
- 用户粘了用户调研 / 访谈纪要让 AI 提炼
- 不符合模式 B 触发条件的所有情况

**特点**：
- 必填项严格检查（功能 / 卖点 缺 → 反问；节点/适配阶段缺失时按当前日期推断）
- 走完整 6 字段输出流程
- 用户敲定后**默认写入 Base**

---

### 模式 B：仿写扩展（宽松流程）

**触发条件**（必须**同时满足**）：
1. 用户**粘了一条完整的方向样本**（含 6 字段或至少含方向标题 + 关键描述）
2. 用户用了仿写关键词（任一）：
   - "类似的再来 X 条"
   - "用这个风格再来几条方向"
   - "仿照这条方向再来"
   - "同款再来 X 条"
   - "换个说法再写"
   - "这种风格的"

**特点**：
- **不追溯必填项**（不反问功能/卖点/适配阶段）
- AI 看样本自己推断
- 出完才问要不要入库（默认偏向"不存"避免污染）

---

### 混合模式（边界，按 A 走）

```
用户："基于已有的拍题精学方向，再来 2 条新角度的"
```

→ 用拍题精学（模式 A 的功能输入）+ 仿样本风格（模式 B 的写法）
→ **入库**（因为有明确功能归属，不是临时仿写）

---

## 模式 B 详细执行流程

### B1：看样本自己推断（不反问）

从用户粘的方向样本里：

| 要推断的项 | 怎么从样本里看 |
|---|---|
| **功能** | 从方向标题 / 解法描述里识别（如"拍题秒出"→ 拍题精学）|
| **卖点** | 从样本的「字段 4 场景问题」+「字段 5 解法」里提取核心卖点 |
| **目标人群** | 直接继承样本里的"目标人群"字段（如果没写，从场景推断学段/特征）|
| **适配阶段** | 样本里有就用；没有则按当前日期或样本语境推断，仍无法命中时默认"日常学习" |
| **数量** | 用户说几条就几条；没说 → **默认 2 条**（少而精）|

### B2：直接出 N 条仿写

风格 / 句式 / 结构都仿样本：
- 主标题：跟样本一样的命名结构（"方向 N：[短句]—[心智锚]"）
- 字段 4：跟样本一样的"画面感场景"结构
- 字段 5：跟样本一样的"反差 + 比喻"结构
- 字段 6：跟样本一样的"以前/现在 对比"结构

**但内容是新角度**：
- 不能跟原样本同一卖点（否则就是同义复述）
- 至少 1 个核心要素（场景 / 人群细分 / 痛点强度 / 解法角度）跟样本不同

### B3：出完才问要不要入库

```
┌────────────────────────────────────┐
│ 2 条方向已出。要存进 Base 吗？     │
│  ⚪ 存（追加到 directions）         │
│  ⚪ 不存（仅供参考）                │
└────────────────────────────────────┘
```

> 默认选项**没有强制项**，让用户自己点。但如果用户没明确指示，倾向"不存"（避免仿写污染方向库）。

如果选"存" → 按主 SKILL.md 的工具调用规则写入 `directions`，并保持模式 B 的宽松推断边界。

## 示例对照

> 示例来自 `功能-拍题精学.md` 的真实业务素材方向，保留真实案例的字段结构和表达密度。

### 模式 A 输入示例

```
用户："拍题精学的开学季方向，3 条"
→ AI 走严格流程：
  - 读卖点库
  - 如有阻塞歧义，用当前运行时可用的交互方式一次性问清关键缺口
  - 出 3 条方向（每条选不同卖点角度）
  - 自查 → 给用户看
  - 用户说"全要" → 写 Base
```

### 模式 B 输入示例

```
用户：「看这条方向：'方向 11：主打识别准确——拍三遍还不对，洋葱一拍就准
       目标人群：频繁使用各种拍题 App、但反复踩坑被迫换工具的中小学生
       场景：用其他 App 拍了三遍题目，总说无法识别；好不容易识别出来了，答案解析还是错的
       解法：洋葱拍题精学一拍精准识别题目，答案经过 AI + 名师教研团队校准，不是 AI 瞎编的“看着像对”
       奇效：拍一次就对一次，不用再拍了改、改了拍，也不用拿错答案交作业被打回来'
       
       类似的再来 2 条」

→ AI 走宽松流程：
  - 推断：功能=拍题精学 / 人群=频繁拍题但不信任结果的学生 / 节点=日常学习
  - 不反问、不读卖点库（直接出）
  - 出 2 条仿写方向（保持“痛点具体 + 洋葱动作 + 可感知结果”的密度，换不同卖点角度）：
    - 方向 X：解析太跳看不懂——洋葱拆细步骤，看到哪步懂哪步
    - 方向 Y：答案对了还扣分——洋葱拆得分点，知道每一步为什么写
  - 出完弹卡：「要存进 Base 吗？⚪ 存 / ⚪ 不存」
```

## 决策树

```
用户输入 →
  ├─ 有"类似/仿写/同款/换说法"等仿写关键词？
  │   ├─ 是 → 用户粘了完整方向样本？
  │   │       ├─ 是 → 模式 B（仿写）
  │   │       └─ 否 → 模式 A（按新建走，但提示用户"如果想仿写，粘一条样本给我"）
  │   └─ 否 → 模式 A
  └─ 模式 A 内部：
      ├─ 用户从零开始 / 给了功能+节点 → 走「输入 A」（见方向生成规则.md）
      ├─ 用户给新卖点 → 走「输入 B」
      └─ 用户粘调研纪要 → 走「输入 C」
```
