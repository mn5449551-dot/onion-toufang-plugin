# 环境自检清单

> onion-help 的跨平台诊断参考。优先用 Python 脚本检查，不把 Mac shell 命令写成唯一入口。

## 快速命令

```bash
python3 skills/onion-help/scripts/setup_wizard.py check
python3 skills/onion-help/scripts/setup_wizard.py ensure
python3 skills/onion-help/scripts/setup_wizard.py bootstrap
python3 skills/onion-help/scripts/setup_wizard.py doctor
```

`setup_wizard.py` 会输出 JSON，核心字段包括：

- `platform.family`: `mac|windows|linux|other`，用于判断后续提示用 Mac 还是 Windows 口径。
- `output_root`: 本地图片输出根目录；可由 `ONION_AD_OUTPUT_ROOT` 覆盖。
- `setup_status_path`: `~/.onion-ad/setup-status.json`。
- `usage_state_path`: `~/.onion-ad/usage-state.json`。
- `first_use`: 本地第一次记录该插件使用状态。
- `auto_bootstrapped`: `ensure` 是否自动做了安全 bootstrap。
- `checks.env_file`: `.env` 是否存在，是否缺 `LAOZHANG_API_KEY` 或 Base token。
- `checks.lark_cli`: 是否能找到 `lark-cli`。
- `checks.python`: Python 版本是否满足。
- `checks.pillow`: Pillow 是否可 import。
- `checks.image_api`: 老张 `GPTImage2 Enterprise / gpt-image-2` 本机并发设置；默认 `ONION_IMAGE_CONCURRENCY=6`、`ONION_IMAGE_FALLBACK_CONCURRENCY=3`，按 `3000 RPM / API key` 和 `100 concurrent requests / API key` 设计，不启用团队级全局锁。
- `checks.scripts`: 关键脚本是否存在。
- `next_actions`: 需要用户处理的下一步，不包含真实密钥值。

## 一键配置（safe ensure/bootstrap）

```bash
python3 skills/onion-help/scripts/setup_wizard.py ensure
```

`ensure` 会先看 `usage-state.json`、`setup-status.json` 和 `.env`。缺任意一个时自动执行 safe bootstrap；都存在时只做 check 并刷新使用记录。bootstrap 会自动做这些安全动作：

- 创建用户配置目录：Mac/Linux 为 `~/.onion-ad`，Windows 为用户目录下 `.onion-ad`。
- 如果 `.env` 不存在，就从插件根目录 `.env.template` 复制一份。
- 创建本地输出目录：默认是系统临时目录下 `onion-ad`，也可以由 `ONION_AD_OUTPUT_ROOT` 指定。
- 写入 `setup-status.json`，让其它 skill 做首启环境门禁时可以快速判断。
- 写入 `usage-state.json`，记录 `first_seen_at`、`last_seen_at`、平台、版本、ready 状态和 bootstrap/check 次数。

ensure/bootstrap 不做这些事：

- 不安装 `lark-cli`。
- 不执行 `lark-cli auth login`。
- 不填 `LAOZHANG_API_KEY` 或其它真实密钥。
- 不自动创建或修改飞书 Base schema。

## 判定口径

| 标记 | 含义 |
|---|---|
| ✅ | 已满足，核心 skill 可继续使用 |
| ⚠️ | 有风险或建议修复，但不一定阻塞 |
| 🔴 | 阻塞项，需要先处理 |

## Mac / Windows 适配

- `platform.family=mac`：可以提示用户用终端编辑 `~/.onion-ad/.env`，并执行 `lark-cli auth login`。
- `platform.family=windows`：提示用户在 PowerShell / CMD 中确认 `lark-cli` 可执行，并编辑 `%USERPROFILE%\.onion-ad\.env`。
- `platform.family=linux`：按 Linux 服务器或 WSL 环境提示。
- `platform.family=other`：只给通用 Python 命令和路径，不猜平台专属命令。

不要把 `which`、`source`、`stat -f`、`chmod`、`vim` 写成唯一修复路径。需要具体命令时，先看 `platform.family` 再给平台对应说明。

## 阻塞项

### 1. `.env`

- 文件不存在：🔴，先运行 ensure。
- `LAOZHANG_API_KEY` 缺失或仍是占位符：🔴，出图会失败。
- `ONION_BASE_APP_TOKEN` 缺失：⚠️，共享脚本有默认 token，但团队环境建议显式配置。

### 2. lark-cli

- `checks.lark_cli.status=missing`：🔴，Base 读写不可用。提示用户安装 lark-cli 并登录。
- 已安装但未登录或 Base 403：🔴，让用户执行登录或联系 Base 管理员加可编辑权限。
- 写 Base 必须走 `--as user`，让飞书系统字段记录真实创建人。

### 3. Python / Pillow / scripts

- Python 低于 3.8：🔴。
- Pillow 缺失：🔴，压缩和导出会失败；提示 `python -m pip install Pillow`。
- 关键脚本缺失：🔴，停止核心流程，先升级插件或修复安装。

### 4. 老张 GPTImage2 Enterprise 并发

- `checks.image_api.provider=GPTImage2 Enterprise`：本插件按老张官转企业版 `gpt-image-2` 线路运行。
- `checks.image_api.local_concurrency`：本机 batch render 最大并发 job 数，默认 6。
- `checks.image_api.fallback_concurrency`：遇到 429 / 5xx / timeout 后当前批次降级重试的并发，默认 3。
- `documented_rpm_per_key=3000`、`documented_concurrent_requests_per_key=100` 是当前设计口径；本期不做飞书队列或团队级全局锁。

### 5. 输出目录

本地输出根目录来自：

```text
ONION_AD_OUTPUT_ROOT 或 系统临时目录/onion-ad
```

Mac 常见为 `/tmp/onion-ad` 或 `/private/tmp/onion-ad`；Windows 常见为 `%TEMP%\onion-ad`。技能回复里应展示实际路径，不要把 `/tmp/onion-ad` 写死为唯一位置。

## 完整体检

`doctor` 可在用户明确要深度排查时使用。它仍然不做付费生图、不写 Base，只返回可读诊断。需要真实 Base 权限检查时，再结合 `lark-base` 或 `lark-cli base` 命令按当前平台执行。

## 总结模板

```text
📋 环境自检报告

✅ 通过：Python / Pillow / shared scripts / output_root
⚠️ 警告：Base token 未显式写入 .env
🔴 阻塞：LAOZHANG_API_KEY 仍是占位符；lark-cli 未安装

下一步：
1. 打开用户目录下 .onion-ad/.env，填 LAOZHANG_API_KEY。
2. 安装 lark-cli 并登录飞书账号。
3. 再运行 setup_wizard.py check。
```
