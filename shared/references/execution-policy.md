# 执行深度策略

这套 skills 要同时保证正确和速度。Agent 先判断当前任务属于哪条路径，再决定是否深入推理或停下追问。

## Fast Path

ID 回查、环境检查、状态查询、压缩、打包、Base 写入、候选排版、读取本地 JSON 都是机械动作。输入已齐时快速执行，不写长推理，不重复解释。

## Gate Path

缺功能/卖点、图片角色不明、配置页缺失、选择页未提交、API key 缺失、上传截图缺失、写 Base 准入不足时必须停下追问或提示修复。正确优先于少问，不能为了省一轮对话猜测入口、版位、父图组、采纳项或是否能付费生图。

## Creative Path

方向生成、文案生成、图片 prompt 生成、批量多样性设计需要更深推理。创意推理必须使用卖点库、真实案例提炼、方向/文案字段规则和 prompt recipes；不能编造产品事实，不能省略参考图编号、字体融合规则、Logo/IP/CTA、版位尺寸、屏幕内容闸门等固定规则。

11-50 套图片 prompt 按 5-10 套一批精写，并维护已用 IP、场景、镜头、动作、视觉隐喻和排版 ledger。速度来自分批和并发渲染，不来自粗糙生成。
