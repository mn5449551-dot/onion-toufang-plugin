# 合规视觉雷区（必读）

> 出图 prompt 写完后**强制扫**这份清单，命中任一 = 立即改 prompt 重出。
> 这里的"雷"主要是应用商店 / 学习机厂商市场审核 + 广告法。

---

## 一、设备识别相关

| ❌ 不能出现 | ✅ 应该这么画 | 原因 |
|---|---|---|
| 手机有明显**刘海屏** | 全面屏 / 不展示听筒区域 | 应用商店要求"不要特定品牌识别" |
| 手机带某品牌 Logo（华为/苹果/小米等）| 无品牌 Logo / 完全虚构机型 | 多渠道投放避免品牌冲突 |
| 平板有明显 iPad / 三星标识 | 通用平板外形 | 同上 |
| 学习机有具体型号（讯飞 X / 学而思 Y）| 通用学习机造型 | 厂商市场避免竞品识别 |

补充固定规则：

- 出现手机、平板、学习机时，不画刘海屏、摄像头开孔、系统 UI 角标或硬件品牌标识；除非画的是洋葱学园 APP 内容，设备本身仍保持抽象通用。
- 出现试卷、课本、作业本、草稿纸时，字面朝向要符合人物视角。人物坐在桌后看卷子时，观众看到的字面可以是倒置的；不要为了让观众看清而让人物“看反面”。

## 二、承诺词 / 数字（文字渲染时注意）

| ❌ 不要写在图里 | ✅ 应该这么写 |
|---|---|
| "提分 30 分" / "保过" / "100% 提分" | "效率提升" / "讲到你会" / 不用数字 |
| "中考必涨" / "高考必过" | "考前充分准备" |
| "全国第 X" / "排名第一" | 不用排名表述 |
| "30 天见效" | 不用绝对时间承诺 |

---

## 三、人物相关

| ❌ 避免 | ✅ 应该 |
|---|---|
| 真实老师 / 名师肖像 | 卡通 IP / 虚构教师形象 |
| 真实学生照片 | IP 形象 / 漫画学生 |
| 教育名人 / 公众人物 | 同上 |

---

## 四、品牌冲突相关

| ❌ 避免 | 原因 |
|---|---|
| 出现竞品名（作业帮 / 小猿 / 题拍拍 等）| 平台审核会拒 |
| 出现竞品 logo / 配色 | 同上 |
| 提到"对比 XX APP" 后写竞品名 | 用"对比其他工具" 即可 |

## 五、内容审核（老张 API moderation）

老张 API 的 `moderation` 参数默认 `auto`。如果 prompt 含敏感内容会拒：
- 政治 / 宗教 / 暴力
- 色情 / 擦边
- 极端情绪表达（"绝望"、"自杀"等）
- 民族 / 种族 相关

教育广告基本不会撞这些，但**强情绪词会直接触发 moderation 拒绝**——不只是合规问题，是 API 不出图。

### 5.1 强情绪词清单（绝对禁用）⛔

prompt 描述里**不能出现**这些词（含 mainChange、seriesPlan、视觉细节段任何位置）：

> 崩溃 / 沮丧 / 痛哭 / 焦虑 / 绝望 / 暴怒 / 受罪 / 折磨 / 抱头痛苦 / 嚎啕 / 撕扯 / 砸东西

也禁画：
- 儿童被强迫 / 受罚 / 被吼 / 被打（任何形式不画）
- 哭泣孩子 + 家长施压组合
- 老师戒尺 / 体罚相关

### 5.2 安全替代词表（按情绪叙事弧分级）

每套图情绪应是**负面 → 思考 → 解决 → 满足**的上升弧线。各阶段允许的词：

| 阶段 | 允许的轻情绪词 | 同义但禁用的强情绪词 |
|---|---|---|
| 痛点起势 | 微皱眉 / 略显困惑 / 茫然 / 稍作思考 / 抓抓头 | ❌ 崩溃 / 焦虑 / 痛哭 |
| 思考过程 | 专注 / 凝神 / 看屏幕 / 略带期待 / 凝神看题 | ❌ 焦头烂额 / 抱头 |
| 解决瞬间 | 恍然大悟 / 眼睛一亮 / 嘴角上扬 / 微微一笑 | ❌ 失控大笑 |
| 满足收尾 | 微笑 / 满意 / 自信 / 给小伙伴击掌 / 露出笑容 | ❌ 狂喜 / 尖叫 |

**判别口诀**："轻情绪 + 上升弧线" → 安全；"强情绪 + 停留在负面" → 拒绝。

### 5.3 改写示例

❌ 原 prompt：「初中生看着试卷崩溃哭泣，焦虑情绪写满脸，桌上散落红叉满屏」

✅ 改写：「初中生看着试卷，微皱眉头略显疑惑，稍作思考；桌上放几处浅色批注；最后表情转为专注微笑，恍然大悟」

## 六、字体相关

文字渲染时注意：
- 默认只用 `assets/font-references/` 的洋葱专属字体参考图，不主动切到外部字体。
- 不要用某品牌专属字体（如苹方非授权场景）。
- 只有用户或维护人明确要求时，才评估思源黑体 / 阿里巴巴普惠体等通用安全字体。

参 `视觉元素规范.md` 的字体章节。

## 七、自查流程

prompt 写完，调 render.py 前：

```
1. 扫 prompt 文本 → 含上面任一关键词 → 改
2. 视觉描述里有"手机/平板/学习机" → 加"通用机型，无品牌标识"
3. 视觉描述里有"试卷/作业/课本" → 检查字面朝向符合人物视角
4. 文字渲染部分（图里要显示的字）→ 扫承诺词 / 数字
5. 涉及人物 → 确认是 IP 形象或虚构人物，不是真实肖像
6. 全过 → 提交 render.py
```

## 八、AI 主动建议

如果用户给的"文案"或"指令"含合规雷区词，AI 应主动提示：

> "你提到的'保过'在应用商店投放可能违规，要不要换成'考前充分准备'？"

不要默默改用户的话（参 6 原则：推断必告知）。
