# 文案常见误区（自查清单）

> 出完每套立刻跑这份自查；命中任一就重写。

---

## 误区 1：同义复述

多套文案看似不同，本质是"快、好、对"的不同说法。

**检查**：把多套文案的"奇效感"那部分（信息流副标 / 双三图最后一句）排成一列，能一眼区分吗？

**反例**：
```
套 1：洋葱一拍识别准确
套 2：洋葱拍题答案更准
套 3：洋葱让你拍一次就对
```
都在说"准"，**重写至少 1-2 套换不同卖点或换场景**。

---

## 误区 2：渠道语气混淆

信息流要"钩子"语气，应用商店/学习机要"直白"语气。

**反例（应用商店文案写成信息流钩子）**：
```
主：看完答案解析更懵了？  ← 太钩子，应用商店用户主动搜索不需要只抛悬念
```

**应该**：
```
主：拍三遍还不对？  ← 直白说问题和卖点入口
```

---

## 误区 3：解法变功能清单

副标 / 短句滑向产品说明书。

**反例**：
```
副：洋葱有 AI 错题本 + 分步解析 + 个性化学习路径...
```
→ 列功能没体验感。

**应该**：
```
副：洋葱拍题一拍就准，答案解析次次对  ← 描述使用感受
```

检查：默认学生视角下，文案要能对应“场景问题 -> 惊艳解法 -> 场景奇效”。至少写清一个产品动作，例如洋葱一拍、拍题精学、分步解析、看不懂再问 AI。

---

## 误区 4：业务越界

写了 APP 没有的功能。

**反例**：
```
短句 2：AI 自动批改作文打分  ← 拍题精学没有这个
```

**检查**：每套文案描述的功能必须在 `shared/knowledge/卖点库.md` 对应功能的卖点矩阵里；未确认的新能力不要写成既有功能。

---

## 误区 4b：把一个功能的场景套到所有功能

**反例**：给洋葱私教班写“一个人做题没人问、拍一下继续追问”，这是把拍题精学的作业卡题场景套到私教班。

**应该**：先理解当前功能。私教班更适合写诊断、分班、个性化计划、30 分钟课堂、AI+真人辅导、学情报告；拍题精学才适合写一拍定位、分步解析、继续追问。

---

## 误区 4c：为了共鸣而编功能

**反例**：没有资料说明有真人老师，却写“真人私教陪你讲到会”；没有资料说明一次解决全部错题，却写“错题本一次清空”。

**应该**：产品动作不清楚时先查知识库或追问；仍不清楚就写得更克制，不能为了共鸣编功能。

---

## 误区 5：产品口径扩张

私教班手册里已有的提分、100%紧贴等表达可以作为业务口径使用；问题在于 Agent 自己凭空扩张承诺。

**反例**：
```
主：30 天包过期中考
副：保证涨分，班级第一
```

**应该**：
```
主：我该怎么学？才能得 A？
副：洋葱私教班先测再学，量身定制提分攻略
```

---

## 误区 6：字数硬卡损失语义

记住：字数是**软约束 / 参考**（合理超出 OK）。如果为了字数硬切让句子没画面感，宁可超字数。

---

## 自查口诀

每套文案出完，按这个顺序扫一遍：
1. **同义复述**：多套并排能区分吗？
2. **渠道语气**：信息流够钩子？应用商店/学习机单图够口语、清楚、有画面感？
3. **解法体验**：副标/短句是体验感不是功能清单？
4. **业务边界**：描述的功能 APP 真的有？
5. **产品口径**：有没有把手册没有给的承诺自己加出来？
6. **通顺自检**：每句大声读一遍像真人说话吗？还是关键词标签云？（参 `规格.md` 的字段和多套生成规则）
7. **主语自检**：有没有把“洋葱”写成奇怪主语，比如“洋葱不懂还能继续问”？如果有，改成“看不懂，再问洋葱 AI”。

7 个都过 → 给用户看；任一不过 → 自己改完再给。

## 标杆 Bad Examples（带改写）

> 凝练自旧投放平台 copy generation prompt 的 6 条 bad_example，**每条都标明违反哪个铁律 + 怎么改写**。

### Bad 1：信息流硬广腔（动词陷阱）

**渠道**：信息流 / **形式**：单图

```
主：立即下载洋葱学园 APP
副：限时优惠快来抢
```

❌ "立即下载 / 限时优惠 / 快来抢" 是动词陷阱，信息流要钩子型不要硬广腔，用户立刻反感划走。

✅ 改写：
```
主：作业讲了 8 遍娃还卡壳
副：试试拍一下让 AI 把每一步讲透
```

### Bad 2：应用商店多图缺"洋葱"解决方主体

**渠道**：应用商店 / **形式**：三图

```
短句 1：题不会做
短句 2：拍一下
短句 3：当场会
```

❌ 图 2 / 图 3 应至少有一处出现"洋葱"或"洋葱学园"，让用户记住是**谁**在解决。

✅ 改写：
```
短句 1：题不会做
短句 2：洋葱拍一下
短句 3：AI 当场讲
```

### Bad 3：默认学生视角被写成家长腔

**渠道**：应用商店 / 学习机单图 / **形式**：单图

```
主：妈妈再也不用担心我的作业了
副：省心陪伴温暖辅导
```

❌ 用户不指定人群时默认学生视角，"妈妈 / 省心 / 陪伴" 会变成家长腔，缺少学生自己的画面感。

✅ 改写（学生视角 + 产品动作）：
```
主：题不会做？拍一下问 AI
副：5 秒出解析，每一步都讲透
```

### Bad 4：多图 3 个孤立短句没图间关系

**渠道**：应用商店 / **形式**：三图

```
短句 1：高效学习
短句 2：智能 AI
短句 3：拍照解题
```

❌ 三句各说各的，**没有并列 / 因果 / 递进 / 互补 任一关系**；连起来读不成故事——浪费 3 倍成本做单图效果。

✅ 改写（因果关系）：
```
短句 1：题不会做
短句 2：洋葱拍一下
短句 3：AI 当场讲
```

### Bad 5：自己加不存在的强承诺

```
主：30 天提分 30 分
副：保过期中考
```

❌ 手册没有给“30 天提分 30 分 / 保过”这种承诺，不能自己加。

✅ 改写为产品动作和学习过程：
```
主：100 道真题练透 7 大考点
副：考前系统刷题，错题自动收录
```

### Bad 6：为了卡字数砍成电报体（关键词堆砌）⭐

**渠道**：应用商店 / **形式**：三图

```
短句 1：大题卡壳
短句 2：洋葱看点
短句 3：下次自己能写
```

❌ 每句都砍成 4 字主谓不全：
- "洋葱看点" 是病句（看什么点？）
- "拿下满分" 没有具体产品动作，且容易变成凭空强承诺
- 三句像关键词标签云不是真人在说话

✅ 改写（口语化通顺，每句 9-12 字仍在软建议内）：
```
短句 1：大题卡壳不知咋下笔
短句 2：洋葱一拍看透得分点
短句 3：下次自己能写出来
```

> **核心心法**：**字数仅作参考，少 / 多都 OK**。唯一硬标准是句子通顺自然——读起来像翻译腔或关键词堆砌（如"大题卡壳 / 洋葱看点 / 拿下满分"这种主谓不全的电报体）就重写。
