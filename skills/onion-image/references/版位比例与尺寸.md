# 版位比例与尺寸

> 事实源是 `../config/channel-placement-rules.json`。本文件只解释字段和决策，不再维护手写版位表，避免和最新规则分叉。

## 规则字段

每个版位至少有这些字段：

| 字段 | 含义 |
|---|---|
| `category` | 应用商店 / 学习机 / 信息流 / 百度 |
| `platform` | 华为、OPPO、vivo、小米、荣耀、读书郎、网易、百度等 |
| `placement` | 具体版位名 |
| `target_size` | 最终交付尺寸，如 `984x554` |
| `render_size` | 传给 `gpt-image-2` 的 `size`，如 `1280x720` |
| `max_file_size_kb` | 导出压缩上限；为空时默认 200KB |
| `directness` | `direct` / `near` / `expand` / `composite` |
| `postprocess` | `resize` / `resize_center_crop` / `outpaint_then_resize` / `layered_composite` |
| `enabled` | 一期配置页是否可选 |
| `disabled_reason` | 置灰原因 |

## 一期开放规则

一期只开放：

- `directness=direct` 或 `directness=near`
- `image_count=1`
- `enabled=true`

这些版位可以直接调用 `render.py --size <render_size>`，再导出到 `target_size`。

一期先置灰：

- `directness=expand`：需要扩图后处理。
- `directness=composite`：需要分层合成。
- 双图 / 三图 / 组图：后续单独做链式或多图版位流程。

## render size 与 target size

不要把渠道目标尺寸直接传给 gpt-image-2。实测 `984x422`、`1080x170` 等目标原尺寸会被接口拒绝，但规则表里的非标准 `render_size` 可以生成，例如：

| render_size | 结果 |
|---|---|
| `1568x672` | 可生成 |
| `1168x1024` | 可生成 |
| `1024x1664` | 可生成 |
| `576x1728` | 可生成 |
| `2480x1024` | 可生成 |
| `2896x1024` | 可生成 |

处理链路：

```text
用户选版位
  ↓
读取 render_size 调 render.py
  ↓
得到原始 PNG
  ↓
按 target_size 做 cover resize / center crop
  ↓
按 max_file_size_kb 压缩成 JPG
  ↓
进入选择页、打包 zip、写 Base 附件
```

## 多版位

一个文案可以选择多个版位。不要把同一张图强行裁成所有版位；每个版位应单独生成 prompt 并按自己的 `render_size` 渲染。

共用项：文案、Logo 选择、CTA 输入、补充说明、界面/功能参考图。

逐图随机项：随机 IP、随机字体参考。
