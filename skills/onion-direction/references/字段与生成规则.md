# 字段定义与方向判断规则

本文件是 `onion-direction` 的方向卡事实源：6 个字段怎么写、输入怎么转成方向、输出前怎么自查。它不负责 Base 命令语法。

## 目录

| 章节 | 用途 |
|---|---|
| 核心模型 | 生成方向时怎么判断角度 |
| 6 个方向字段 | 每条方向必须输出的字段语义 |
| 辅助字段 | 写 Base 时额外需要的字段 |
| 输入转换 | 用户给不同材料时怎么处理 |
| 决策检查表（内部） | 交给用户前的质量门槛 |

## 核心模型

每条方向 = `卖点 x 场景 x 细分人群 x 切入角度`。

| 维度 | 判断标准 | 好例 |
|---|---|---|
| 卖点 | 本条方向真正使用的产品能力 | 拍题秒出、分步解析、哪步不懂点哪步 |
| 场景 | 有时间、地点、行为的具体学习瞬间 | 晚 9 点书桌前赶数学作业 |
| 细分人群 | 带行为特征的人，不是泛泛学段 | 作业量大、对等待零容忍的初二学生 |
| 切入角度 | 叙事入口 | 场景、痛点、效果对比、情感、权威、反差、性价比 |

创意多样性来自 4 维组合，不来自同义改写。任意两条方向至少 2 维不同；3 条及以上至少覆盖 3 类切入角度；5 条及以上至少覆盖 4 类切入角度，同一角度最多 2 条。

### 7 类切入角度

| 角度 | 什么时候用 | 关键判断 |
|---|---|---|
| 场景切入 | 用户场景很具体 | 有时间、空间、人物和动作，能直接拍出来 |
| 痛点切入 | 用户痛点强 | 只打一件具体困扰，不堆多个痛点 |
| 效果对比 | 想表达前后变化 | 写行为或能力变化，不写提分承诺 |
| 情感共鸣 | 家长/学生情绪真实 | 情感必须和功能解决链条有关 |
| 权威背书 | 有教研、师资或历史事实 | 背书要具体，避免第一、唯一、顶尖等绝对化 |
| 反差冲突 | 需要提高停留 | 反差要可信，并在解法字段解释清楚 |
| 性价比 | brief 里有价格/权益 | 价格和限时信息必须真实，不编优惠 |

来源参考：`business-knowledge/references/for-stage-2b/angles.md`。本 Skill 只吸收角度判断，不强制输出额外 `angle` 字段，因为当前 Base directions 表没有该字段。

## 6 个方向字段

### 1. 素材方向

用途：一句话点出投放主题，让运营能判断这条方向主打什么。

写法：`方向 N：[核心卖点短句]——[心智锚定或落地动作]`

强例：

> 方向：主打“识别准确”——拍三遍还不对，洋葱一拍就准

> 来源提炼：`功能-拍题精学.md` 方向 11；真实复盘里“识别准确”也是应用商店和多渠道稳定方向。

要求：

- 像投放主题，不像功能清单。
- 主标题尽量短，破折号后补心智锚定。
- 不写提分、保过、100% 等承诺。

### 2. 目标人群

用途：定义这个方向说给谁听。

强例：

> 时间紧、作业量大、对“等待”零容忍的学生，尤其周中晚间赶作业群体。

要求：

- 写行为和情境特征，不只写“K12 学生”“中等生”。
- 能让下游文案和画面知道该拍谁。
- 30-50 字是参考，不硬卡。

### 3. 适配阶段

用途：说明这个方向适合哪个投放节点或学习阶段。

常用值：日常学习 / P0-寒假 / P0-开学 / 期中前 / 期末前 / 暑期 / 其他。

要求：

- 必须呼应场景问题。
- Base 是单选字段，入库时用选项名。
- 用户给“开学季”时通常映射到 P0-开学。

### 4. 1 能解决用户在“具体哪个场景里的哪个问题”

用途：写痛点画面。读完应该能直接拍成 3 秒短视频。

强例：

> 晚上 9 点，作业还剩五道数学大题。打开各种拍题 App，拍了一下就开始转圈、加载、正在生成解析，干等了快 5 分钟。题还没开始做，注意力已经散了，只想赶紧抄完睡觉。

要求：

- 有时间、空间、人物、行为。
- 有具体阻塞，不写“作业难”“学习压力大”这种空话。
- 情绪可以有，但避免强刺激词。

### 5. 2 能带来什么不一样的“一听很惊艳”的解法？

用途：写洋葱的差异化解法。

强例：

> 洋葱拍题精学在拍完题的瞬间就开始逐步呈现解析，不是等全部生成完才给你看，而是像老师在黑板上实时写板书，边讲边推进。

要求：

- 必须出现具体功能、动作和使用体验。
- 最好有“不是 A，而是 B”的反差。
- 不要堆功能名；要写为什么它对当前场景有用。

### 6. 3 因此带来了哪个场景下的什么“奇效”？

用途：写同一场景下的前后变化。

强例：

> 以前五道题光等加载就要十几分钟，现在拍完即看、看完即写。作业流程从“拍-等-看-写”变成“拍-看-写”，省下的不只是时间，是做题时不断掉的专注力。

要求：

- 呼应字段 4 的场景。
- 写“以前/现在”或等价前后对比。
- 奇效必须可感知，不写泛泛的“更高效、更省事”。

## 辅助字段

写 Base 时还需要：

| 字段 | 规则 |
|---|---|
| 功能 | 从用户 brief、样本或方向标题识别 |
| 卖点 | 字符串数组，来自 `shared/knowledge/卖点库.md` 或用户明确的新卖点 |
| 状态 | 新建默认 `待用` |

`适配阶段` 是 Base 字段，不等于完整时间节点。遇到开学、寒暑假、期中期末、备考、月考或具体日期时，先读 `时间节点策略.md`，把节点语义转成字段 4/5/6 的场景和解法。

不要传自动字段：方向ID、创建时间、创建人、最后更新时间。

## 输入转换

### 功能 + 节点 + 数量

例：`拍题精学开学季 3 条方向`

处理：

- 功能 = 拍题精学。
- 节点 = P0-开学。
- 数量 = 3。
- 从卖点库挑互补卖点，不把 3 条都挤在同一痛点上。

### 新卖点或新功能描述

先判断它是否已经存在于卖点库。不存在时，不要直接编成既有能力；先用当前上下文确认它解决的场景、人群和差异点，并提示维护人后续沉淀到卖点库。

### 节点和时间

用户给出寒假、暑假、开学季、期中、期末、月考、备考或具体日期时，按 `时间节点策略.md` 判断。当前日期或用户投放排期落在表格空档时，不要假装命中；用户给了节点就按用户节点生成，用户没给节点就选日常学习或下一节点预热，并说明这是推断。

### 用户研究 / 访谈纪要

优先提取真实痛点原话和高频场景，再反查能解决它的卖点。字段 4 可以保留研究材料里的具体表达，不要把真实话语抽象成“学习困难”。

### 仿写样本

样本完整且用户说“类似再来”时，不反问。继承样本的功能、人群、阶段和结构，但必须换卖点或场景角度，避免同义复述。

## 决策检查表（内部）

Use this checklist privately. Only summarize issues to the user if asked why.

- **换品牌检查**：把字段 5 里的“洋葱”换成“某 APP”，如果仍成立，说明差异化不够，重写。
- **画面感检查**：字段 4 是否有时间、空间、人物、行为；缺一项就补。
- **同义复述检查**：把所有字段 6 单独排列，能否一眼看出不同收益。
- **业务边界检查**：是否写了洋葱当前没有的能力。
- **合规检查**：是否出现包过、保证、提分 N 分、100%、唯一、第一等承诺。
- **阶段呼应检查**：适配阶段、场景问题和奇效是否在同一学习节点上。
